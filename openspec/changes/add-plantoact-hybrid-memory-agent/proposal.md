# Change: Add Plan-to-Act Agent Platform

## 背景
当前目标不是做一个单点 agent，而是做一个可扩展的 agent 平台底座：顶层负责规划与编排，底层负责执行、记忆、检索、异步事件、前端展示与跨平台消息适配。

## 目标
1. 将传统 ReAct 改造成 `Plan-to-Act`。
2. 将每个计划步骤拆成可独立执行的 `micro-ReAct` 循环。
3. 引入短期记忆与长期记忆，长期记忆可通过 `Mem0` 或兼容实现接入。
4. 构建混合检索层：稳定资料走 hybrid RAG，动态且结构化资料走 LLM wiki。
5. 增加前端控制台、事件总线、存储分层、可观测性与权限边界。
6. 支持不同外部平台的消息适配与统一消息模型。
7. 增加 SSE / 流式输出能力与渐进式前端渲染。
8. 增加端到端日志追溯、证据链追踪与审计能力。
9. 增加模型生成内容质量评估与门禁能力。
10. 增加统一工具调用契约与能力注册。
11. 增加人格设定、角色切换和多角色工作模式。
12. 增加工具与插件注册治理、执行权限和能力声明。
13. 增加安全边界、密钥管理、沙箱执行和脱敏能力。
14. 增加模型路由、降级和成本治理能力。
15. 增加 human-in-the-loop 审批与人工接管能力。
16. 增加知识生命周期管理，包括过期、回滚、删除和保留策略。
17. 增加平台级 SLO、容量与成本预算约束。
18. 增加部署、恢复、断点续跑和灾备要求。
19. 为后续 RL / evaluation harness / safety gating 留出接口。

## 非目标
1. 不在本次变更中训练基础模型。
2. 不强制绑定单一向量库、数据库或工作流框架。
3. 不在本次变更中实现完整生产级权限体系。
4. 不在本次变更中绑定单一云厂商或单一区域部署方案。

## 成功标准
1. 任意任务都能先生成计划，再逐步执行。
2. 每个步骤都能独立观察到工具调用、状态与结果。
3. 记忆和检索结果可追溯来源。
4. 动态知识更新后能快速反映到 LLM wiki，不污染稳定文档索引。
5. 来自不同平台的消息能够被统一接入、路由和回传。
6. 长任务和生成过程能够通过 SSE 或等价流式通道逐步呈现。
7. 任意一次关键响应都能够追溯到消息、模型、提示词、检索证据与工具调用链路。
8. 模型输出质量能够被离线评测、在线抽检和阈值门禁约束。
9. 工具和插件调用在运行前能够完成权限校验、能力声明校验和风险控制。
10. 人格和角色切换能够影响表达风格、工具偏好与工作边界，但不会破坏审计和安全约束。
11. 高风险动作能够触发人工审批或人工接管。
12. 知识条目支持过期、版本回滚、删除和保留策略。
13. 平台能够对延迟、吞吐、失败率和成本预算进行观测与约束。
14. 服务异常后任务能够恢复、回放或断点续跑。
15. 评测与学习闭环有明确门禁，避免直接在线放大错误。
