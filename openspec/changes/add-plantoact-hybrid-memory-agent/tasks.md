# Tasks

## Phase 0: Contract First
1. 定义 agent 状态模型、计划模型与 step 结果模型。
2. 定义统一消息模型与平台适配器接口。
3. 定义人格/角色模型、切换规则和默认行为边界。
4. 定义流式输出事件模型与 SSE 传输规范。
5. 定义工具与插件注册表、能力声明和权限模型。
6. 定义工具调用契约、输入输出 schema 和副作用描述。
7. 定义 trace id、审计日志结构与证据链记录规范。
8. 定义模型路由策略、备用模型和成本统计接口。

## Phase 1: MVP Online Loop
9. 实现前端控制台的基础视图、任务追踪入口、角色切换入口和流式渲染。
10. 实现顶层 plan 生成与 step 调度。
11. 实现 step 内 micro-ReAct 执行循环。
12. 接入事件总线与异步 worker。
13. 拆分任务状态、事件日志、文档、向量、wiki 与审计存储。
14. 实现跨平台请求到响应的端到端日志追溯。

## Phase 2: Knowledge and Retrieval
15. 接入短期记忆与长期记忆接口。
16. 实现 hybrid RAG 路由、检索、重排与引用输出。
17. 实现 LLM wiki 的写入、更新、过期与查询接口。
18. 定义知识条目的 TTL、版本、软删除和归档策略。

## Phase 3: Governance and Safety
19. 实现模型输出质量评测管线与评分维度。
20. 接入线上抽样评估、人工复核和质量门禁。
21. 实现评测 harness 与轨迹采集。
22. 为工具执行补齐超时、重试、幂等和风险等级策略。
23. 实现密钥管理、脱敏策略与受限执行环境接口，并支持本地 model-config 文件优先、env 兼容回退。
24. 实现人工审批、人工接管和恢复执行流。
25. 定义在线与异步链路的 SLO、预算与告警阈值。

## Phase 4: Resilience and Optimization
26. 实现任务恢复、死信处理、重放与灾备演练接口。
27. 为 RL 预留 reward、policy log 与安全门禁接口。
